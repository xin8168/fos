## 概述

为FOS项目各核心模块创建了全面的性能基准测试框架，使用 Criterion 0.5 进行性能测试和基准对比。

## 完成内容

### 1. 创建性能测试框架
- 位置: `src/benches/`
- 使用 Criterion 0.5 框架
- 独立的 workspace 成员，避免与其他模块编译冲突

### 2. 实现的基准测试

#### 2.1 缓存性能测试 (`benchmark_cache`)
测试本地缓存的各项操作性能：

| 测试项 | 数据规模 | 测试内容 |
|------|---------|---------|
| `set` | 100, 1k, 10k | 缓存写入性能 |
| `get` | 100, 1k, 10k | 缓存读取性能 |
| `exists` | 100, 1k, 10k | 键存在性检查 |
| `keys` | 100, 1k, 10k | 获取所有键 |

#### 2.2 分布式锁性能测试 (`benchmark_lock`)
测试分布式锁的各项操作性能：

| 测试项 | 测试内容 |
|------|---------|
| `try_lock` | 非阻塞锁获取性能 |
| `unlock` | 锁释放性能 |
| `is_locked` | 锁状态查询性能 |
| `get_lock` | 获取锁信息性能 |

#### 2.3 快照创建性能测试 (`benchmark_snapshot`)
测试回滚快照的创建性能：

| 测试项 | 数据规模 | 测试内容 |
|------|---------|---------|
| `create_snapshot` | 10, 100, 500 | 不同数据大小的快照创建 |

#### 2.4 并发操作性能测试 (`benchmark_concurrent_operations`)
测试系统的并发处理能力：

| 测试项 | 并发数 | 测试内容 |
|------|--------|---------|
| `concurrent_cache_set_get` | 4, 8, 16 | 并发缓存的读写操作 |

#### 2.5 内存和序列化性能测试 (`benchmark_memory`)
测试序列化和内存操作性能：

| 测试项 | 数据规模 | 测试内容 |
|------|---------|---------|
| `json_serialize` | 10, 100, 500 | JSON序列化性能 |
| `json_deserialize` | 10, 100, 500 | JSON反序列化性能 |

### 3. 技术实现细节

#### 3.1 异步性能测试
使用 `tokio::runtime::Runtime` 在同步 benchmark 中执行异步操作：
```rust
let rt = tokio::runtime::Runtime::new().unwrap();
b.iter(|| {
    rt.block_on(async {
        // 异步代码
    })
});
```

#### 3.2 正确的 API 使用
- 缓存操作使用三参数 `set(key, value, ttl)` 格式
- 移除了不存在的方法调用（如 `remove`）
- 使用正确的数据类型和参数

#### 3.3 性能测试组织
- 使用 `benchmark_group` 组织相关测试
- 使用 `BenchmarkId` 标识不同规模的数据测试
- 独立的测试函数，易于扩展和维护

## 预期性能指标

这些基准测试将提供以下性能指标：

1. **缓存吞吐量**: 操作/秒 (ops/sec)
2. **锁操作延迟**: 平均/最大延迟时间
3. **快照创建时间**: 不同数据规模的创建时间
4. **并发扩展性**: 在不同并发数下的性能表现
5. **序列化性能**: JSON 序列化的吞吐量和延迟
