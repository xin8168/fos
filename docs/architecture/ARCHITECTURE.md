# FOS 系统架构文档

## 架构概述

FOS（Framework of Operations System）是一个确定性执行主板，用于安全、可靠地控制数字/物理设备。

---

## 核心设计原则

### 1. 确定性执行
- 所有操作必须有明确的执行步骤
- 不做任何推理或猜测
- 只执行「成功过」的任务

### 2. 安全隔离
- 所有操作先在沙箱预执行
- 失败自动回滚
- 零风险穿透

### 3. 明文协议
- 所有指令使用中文明文
- 所有输出中文化
- 100%可读可追溯

## 模块职责

### 核心模块（不可修改）

| 模块 | 职责 | 铁律 |
|-----|------|------|
| Gateway | 格式校验、协议转换 | 不做规则判断 |
| Validator | 权限校验、红线拦截 | 不执行操作 |
| Bus | 任务调度、步骤执行 | 不做规则判断 |
| Memory | 成功事件存储 | 不存储失败 |
| Audit | 拦截日志记录 | 不存储成功 |

### 扩展模块

| 模块 | 职责 | 铁律 |
|-----|------|------|
| Sandbox | 隔离预执行 | 不访问真实设备 |
| Skills | 执行步骤管理 | 不做规则判断 |
| MCP | 设备协议转换 | 不做规则判断 |
| Rollback | 失败回滚 | 不执行正向操作 |

---

## 数据流

```
用户指令 → Gateway校验 → Validator检查 → Bus调度
    ↓
Sandbox预执行 → Skills执行 → MCP转换 → 设备执行
    ↓
结果校验 → Memory存储(成功) / Audit记录(失败)
    ↓
返回用户
```

## 安全架构

### 九大铁律

1. **链路唯一铁律** - 所有指令顺序通过
2. **主板不可变铁律** - 核心5模块不可修改
3. **沙箱隔离铁律** - 失败0%穿透
4. **SKILLS验证铁律** - 未验证100%拦截
5. **MCP管控铁律** - 原生协议100%拦截
6. **失败必回滚铁律** - 回滚100%成功
7. **明文输出铁律** - 中文100%
8. **幂等执行铁律** - 结果100%一致
9. **审计留痕铁律** - 操作100%记录
