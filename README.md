## 🎯 项目简介

**FOS神经元控制器**是一个确定性执行控制框架，专为AI能力延展而设计。

在AI向AGI演进的过程中，大模型缺乏的不是"思考能力"，而是"执行可靠性"。FOS解决的核心问题是：**让AI的决策能够100%被正确执行，让具身智能的每个动作都精确可控**。

### 核心应用场景

|---------|------|---------|
| **大模型能力延展** | 将LLM的推理能力延伸到真实世界的执行 | 决策→执行闭环 |
| **脑机能力延展** | BCI信号到设备控制的确定性映射 | 信号→动作映射 |
| **具身智能控制** | 机器人/无人机等复杂设备的精确控制 | 意图→动作执行 |
| **AI Agent执行** | 多步骤复杂任务的可靠执行 | 规划→验证→回滚 |

## 🔑 核心特性

### 1. 确定性执行

- **纯规则执行引擎**: 拒绝任何推理和猜测，每一步都有明确规则
- **六维锚定协议**: 意图信号标准化（事件、步骤、判断、校验、地点、主体）
- **零模糊执行**: 动作执行结果100%可预测

### 2. 安全隔离

- **脊髓反射机制**: 危险动作直接拦截，不上传到大脑
- **三层神经网络**: Gateway(感觉) → Validator(脊髓) → Bus(运动)
- **失败零穿透**: 沙箱隔离确保失败不级联

### 3. 完整可追溯

- **全链路神经记录**: 从意图信号到执行完成的完整追踪
- **毫秒级响应**: 实时状态监控与反馈
- **100%可审计**: 所有执行动作可回溯

### 4. 具身控制

- **MCP设备协议**: 统一的设备控制接口
- **Skills技能库**: 可复用的技能定义
- **离线容错**: 网络中断时本地执行

## 🚀 快速开始

### 安装

```bash
# 克隆项目
git clone https://github.com/fos-platform/fos.git
cd fos

# 构建
cargo build --release
```

### 神经信号输入示例

```bash
# 方式1: CLI输入
fos exec --锚定 '{
  "事件": "移动到指定位置",
  "步骤": ["计算路径", "执行移动", "验证位置"],
  "判断逻辑": "路径无障碍",
  "校验标准": "到达目标点",
  "地点": "机器人A",
  "主体": "LLM控制器"
}'

# 方式2: API输入
POST /api/v1/execute
{
  "anchor": { ... },
  "options": {
    "timeout": 30,
    "rollback": true
  }
}
```

### Docker部署

```bash
# 构建镜像
docker build -t fos:latest .

# 启动神经元控制器
docker run -p 8080:8080 fos:latest

# 查看神经网络状态
curl http://localhost:8080/health
```

## 🛡️ 安全铁律

FOS神经元控制器遵循四条不可违背的安全铁律：

1. **信号链唯一**: 所有意图信号必须通过感觉→脊髓→运动三层
2. **脊髓不可绕过**: 脊髓神经校验不可跳过
3. **反射弧隔离**: 危险动作反射执行，不上传大脑
4. **反馈必闭环**: 执行结果必须反馈到控制层
